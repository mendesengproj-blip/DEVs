kappa  = beta*a0                      -> [m s^-2]

Pi^(A) = (2/3) kappa (theta')^2       (paper Eq. ~258)
  with (theta')^2 dimensionless => Pi^(A) has units [m s^-2]
8 pi G * Pi^(A)                       -> [m^3 kg^-1 s^-2]*[m s^-2]

Required for nabla^2(Psi-Phi)         -> [s^-2]
MISMATCH factor                        = [kg m^-4 s^2]
This is the structure of (density / length^2 * time^2);
equivalently, a missing factor of 4 pi G rho * (1/length^2)
after multiplying by [m^2/s^2]/[m^2/s^2].
=> The literal source S = 8 pi G (2/3) kappa (g_N/a0)^2 is
   DIMENSIONALLY INCONSISTENT with a Poisson equation.

   The paper derivation collapses (theta')^2 -> sqrt(2 X0) g_N

   This carries [s^-2] and reduces, for a point mass,
   to the Green-function integral that produces Eq. 48.

(1) Dimensional analysis (Task 1) shows the source S as written
    in the paper around line 271 is DIMENSIONALLY INCONSISTENT
    when read literally.  Units of 8 pi G * (2/3) kappa (g_N/a0)^2
    are [m^4 kg^-1 s^-4], not [s^-2].
(2) The numerical 10^8-10^13 % residuals seen in eta_verification.py
    are therefore NOT a bug in the FD solver — they are the
    quantitative expression of (1).  This is an
    IMPLEMENTATION-vs-DERIVATION mismatch, not a code bug.
(3) The analytic formula eta-1 = (2/3) beta / sqrt[y(1+y)]
    is derived using a *Green's function for a point source* and
    the deep-MOND identification (theta')^2 ~ sqrt(2 X0) g_N.
    Both ingredients are valid only in the asymptotic point-source,
    deep-MOND regime (g_N << a0).  Outside this regime the
    formula is at best an interpolation written by hand.

CLASSIFICATION:  FORMULA_VALID_IN_LIMIT

  concentrated) source AND deep-MOND (g/a0 << 1).
  Outside this regime Eq. 48 should be treated as an
